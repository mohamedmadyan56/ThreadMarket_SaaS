# Project Structure

The project follows a modular and scalable folder structure to keep the codebase organized and maintainable.

```
src/
├── controllers/     # Business logic for handling requests and responses.
├── helpers/         # Reusable helper functions used across the application.
├── middlewares/     # Express middlewares (authentication, authorization, error handling, etc.).
├── routers/         # API route definitions and endpoint registration.
├── schema/          # Database schemas, models, and ORM-related definitions.
├── types/           # TypeScript interfaces, types, and custom type definitions.
├── utils/           # Utility functions and third-party service integrations.
└── validators/      # Request validation logic for controllers and routes.
└── Classes/         # Making reusable classes of every Object Existed in system.
```
